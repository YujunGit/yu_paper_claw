# Daily Paper Digest — 2026-03-20

## Theme
Pushing world models and structured generation beyond short-lived pixel history: persistent 3D state, geometry-aware memory, and executable decomposition.

## Short overview
Today is a **solid but not crowded** day. The strongest paper is **Beyond Pixel Histories: World Models with Persistent 3D State** because it directly attacks a real bottleneck: most interactive world models still store history as images and hope geometry emerges implicitly. **WorldStereo** is weaker as a world model paper than its title suggests, but still useful because its dual-memory design is a concrete mechanism for enforcing multi-view consistency without retraining a giant video model from scratch. **CityGenAgent** is more adjacent than direct, yet worth keeping because its block/building program split is a credible example of decomposition that is executable, editable, and tied to spatial reward design rather than prompt theatrics.

I skipped several nearby papers that were mostly benchmarks, broad surveys, or generic controllable-generation packaging. These three had the best combination of mechanism, transferability, and conceptual clarity.

   - Verdict: **Useful**

3. **Imagine a City: CityGenAgent for Procedural 3D City Generation**  
   - Verdict: **Useful**  
   - Role: **Adjacent inspiration / citation material**

## Most relevant to current work
**Beyond Pixel Histories: World Models with Persistent 3D State** is the clear best hit. The important point is not just “it uses 3D,” but that it treats persistent latent state and camera-query rendering as the core memory interface, which is much closer to a serious world-model stance than short-context autoregressive video prediction.

## Novelty / framing / baseline impact
- **Framing impact:** High. Today’s best paper reinforces a useful distinction: a world model should carry forward structured state, not merely recycle recent frames.
- **Related-work impact:** Strong for persistent 3D memory, geometry-aware controllable generation, and executable decomposition in 3D content generation.
- **Novelty threat:** Low to moderate. PERSIST is the only one that feels like a direct conceptual neighbor to structured world-model work.
- **Caution:** WorldStereo is valuable as a geometry-consistency mechanism, but it is still fundamentally a camera-guided VDM enhancement. CityGenAgent is interesting for decomposition, but its reward pipeline leans heavily on LLM judging and hand-designed priors.

## One-paragraph takeaway
The useful common thread today is that explicit structure only matters when it takes over a real job. **PERSIST** uses a persistent 3D world-frame so memory lookup becomes geometric rather than historical. **WorldStereo** uses global point-cloud memory plus local stereo-style correspondence memory to keep generated trajectories mutually reconstructible. **CityGenAgent** uses executable intermediate programs because city generation genuinely benefits from separating coarse spatial layout from fine architectural realization. The broad lesson is good: explicit structure earns its keep when it changes retrieval, control, editing, or rollout stability—not when it is only decorative nomenclature.
